# node-apollo-graphQL


Common Features
Single Endpoint: Unlike REST, which uses multiple routes, GraphQL uses a single endpoint for all data interactions.
Efficient Data Fetching: Clients can request exactly what they need, preventing over-fetching and under-fetching.
Universal Connectivity: It can connect to any data source, such as MongoDB, PostgreSQL, or SQLite via ORMs like Prisma or Sequelize.
Integration Support: Easily integrates with popular web frameworks like Express or as part of a MERN stack (MongoDB, Express, React, Node).
